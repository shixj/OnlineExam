# Online Exam MVP 验收清单

## 自动化验证

- 运行 `npm --workspace apps/server run test`
- 运行 `npm run build`

期望结果：
- 后端测试通过，覆盖登录鉴权、导入、发布、续做、错题流程
- 后台、练习端、后端服务都能成功构建

## 人工验证：导入与发布

1. 使用 `npm --workspace apps/server run dev` 启动后端服务
2. 打开后台页面，用 `admin / admin123` 登录
3. 下载 Excel 模板
4. 上传一个无效工作簿，确认能看到按行定位的错误信息
5. 上传一个有效工作簿，确认能看到预览数据
6. 确认导入并发布题库
7. 确认题库列表中显示为 `published` 状态

## 人工验证：登录与练习

1. 打开练习端页面，用 `student1 / 123456` 登录
2. 确认已发布题库能正常展示
3. 打开题库详情并开始顺序练习
4. 提交一道题后刷新页面，或重新进入该题库
5. 确认系统会续做原会话，而不是新建重复会话
6. 完成整套练习，确认结果页正常出现
7. 打开历史记录，确认已完成的练习记录已写入

## 人工验证：错题库

1. 开始一轮练习，并故意答错至少一题
2. 打开错题列表
3. 确认答错的题目已进入错题库
4. 启动错题专练
5. 确认第一题来自未掌握错题集合

## 人工验证：手机布局

基准视口：
- 390 x 844 竖屏
- 844 x 390 横屏

检查项：
- 顶部导航按钮可正常点击，且没有重叠
- 首页卡片换行自然，内容清晰可读
- 做题页题干不会被裁切
- 选项按钮尺寸适合手指点击
- 结果页、错题页、历史页没有横向溢出

## 人工验证：iPad 布局

基准视口：
- 820 x 1180 竖屏
- 1180 x 820 横屏

检查项：
- 首页能利用更宽空间，而不是简单把手机单列拉宽
- 题库详情页可以舒适显示题库信息和分类区域
- 做题页在空间足够时，题干区和选项区能并排显示
- 横屏下的选项网格保持可读且便于点击
- 横竖屏切换后，已选答案和当前题目上下文不会丢失

## 签收标准

- 导入校验和题库发布流程通过
- 用户登录、续做、历史记录、错题流程全部通过
- 手机和 iPad 的横竖屏检查通过，不出现裁切、重叠或不可读控件
