## Why

当前仓库的练习端是基于 React 的 H5 页面，适合浏览器和内部测试，但不能直接作为正式微信小程序交付。为了保留现有 H5 投入并支持正式微信小程序路线，需要新增一套独立的原生微信小程序客户端，并复用现有后端 API 与题库、做题、错题、历史记录能力。

## What Changes

- 新增 `apps/miniapp` 原生微信小程序工程骨架，作为与现有 H5 并行的第二个练习端。
- 建立小程序全局配置、登录态存储、请求封装、页面导航和基础 UI 规范。
- 实现首批核心页面骨架：登录、首页题库列表、题库详情、顺序练习、错题列表、历史记录、结果页。
- 复用现有后端账号密码登录与练习 API，不在第一阶段引入微信开放能力绑定。
- 补充小程序本地开发、真机调试、构建接入说明，方便后续正式接入微信发布流程。

## Capabilities

### New Capabilities
- `native-miniapp-shell`: 原生微信小程序工程结构、全局配置、请求层、登录态与页面导航能力。
- `native-miniapp-practice-client`: 原生小程序端的登录、题库浏览、练习、错题、历史、结果等核心练习能力。
- `native-miniapp-device-support`: 原生小程序端在手机与 iPad 上的基础布局、触控与真机调试要求。

### Modified Capabilities
- `user-auth-and-access`: 现有“练习端登录”要求扩展为同时支持 H5 练习端与原生微信小程序端复用同一套账号密码认证接口。

## Impact

- 新增代码目录：`apps/miniapp`
- 影响现有后端认证与练习 API 的客户端接入方式与调试说明
- 需要补充小程序开发者工具工程配置和项目文档
- 不替换现有 `apps/app`，而是与其并行维护
